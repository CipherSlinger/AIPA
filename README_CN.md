# AIPA — AI 个人助手

> 你的桌面智能伙伴。随问随答，随需随做——由 Claude 驱动。

AIPA 是一款基于 Electron + React 构建的桌面 AI 个人助手。它的目标不仅仅是一个聊天窗口，而是一个真正驻留在你桌面上的智能代理：它能理解你的文件与系统环境，编写并执行代码，浏览网页，从头到尾地完成现实世界中的任务。底层通过驱动 [Claude Code](https://claude.ai/code) CLI 作为执行引擎来实现这一切。

## 产品愿景

大多数 AI 工具是"回答机器"。AIPA 要做的是一个**行动者**。

- 你描述你要做什么，它负责想清楚怎么做。
- 它可以读取、创建、修改你本地的任何文件。
- 它可以运行 Shell 命令和脚本。
- 它可以访问网页、抓取信息。
- 它记住你的历史会话，随时继续上次的工作。
- 它将每一步操作透明呈现，你始终掌握最终决定权。

Claude Code CLI 是引擎，AIPA 是驾驶舱。

## 功能特性

- **对话面板** — 通过 stream-JSON 协议与 Claude 进行结构化对话，支持工具调用可视化
- **终端面板** — 基于 xterm.js 的完整 PTY 终端，直接运行 Claude Code，支持进程退出后一键重连和错误恢复；当原生 PTY 模块不可用时（如 Windows 缺少 C++ 构建工具）自动切换到基本 Shell 模式
- **历史会话** — 浏览并恢复 `~/.claude/projects/` 中的历史对话，支持搜索高亮、骨架屏加载和当前会话高亮
- **文件浏览器** — 在侧边栏中浏览工作目录
- **设置面板** — 配置 API 密钥、模型、字体、工作目录、CLI 参数及 MCP 服务器
- **引导向导** — 首次启动时的图形化引导配置流程
- **消息评分** — 通过右键菜单对 AI 回复进行点赞 / 踩评价
- **消息右键菜单** — 右键点击任何消息可快速操作：复制、评分、回滚到检查点
- **思考块** — 可折叠的扩展思考过程展示
- **Token 用量与费用** — 状态栏实时显示上下文用量条、每轮 Token 计数和累计会话费用
- **权限提示卡片** — 以友好方式展示工具调用权限请求（Bash、写文件、编辑文件、网页访问等），支持一键允许 / 拒绝
- **Slash 命令弹窗** — 输入框中输入 `/` 弹出可用命令列表
- **@提及弹窗** — 通过 `@` 引用文件或上下文
- **计划卡片** — 在对话中以结构化卡片展示执行计划
- **图片粘贴** — 直接将图片粘贴至对话输入框
- **文件拖放** — 从系统文件管理器拖放文件到对话区域；图片自动附加，其他文件通过 `@路径` 引用
- **对话导出** — 通过工具栏按钮或 `Ctrl+Shift+E` 将对话导出为 Markdown、HTML 或 JSON；HTML 导出生成自包含的样式化文件，方便分享
- **命令面板** — `Ctrl+Shift+P` 打开模糊搜索面板，快速访问所有操作和命令
- **键盘快捷键** — `Ctrl+N` 新建对话、`Ctrl+B` 切换侧边栏、`Ctrl+`` 切换终端、`Ctrl+L` 聚焦输入框、`Ctrl+,` 设置、`Ctrl+Shift+E` 导出、`Ctrl+F` 对话内搜索、`Ctrl+/` 快捷键速查、`Ctrl+Shift+N` 切换笔记面板
- **会话自动标题** — 首次对话后自动生成会话标题，侧边栏实时刷新
- **系统提示** — 为每次会话注入自定义系统提示词
- **智能自动滚动** — 自动滚动尊重用户意图；阅读较早消息时会显示"回到底部"按钮
- **生成计时器** — 实时显示 Claude 工作时长
- **设置关于页** — 版本信息、外部链接、快捷键参考、运行时信息、恢复默认设置
- **虚拟消息列表** — 使用 `@tanstack/react-virtual` 实现 100+ 消息时的流畅性能
- **RAF 节流流式传输** — 文本增量通过 `requestAnimationFrame` 批量处理，最小化 UI 开销
- **对话内搜索** — `Ctrl+F` 搜索当前对话，支持匹配高亮和前后导航
- **输入历史** — 按上/下方向键在历史发送记录中切换，类似终端 Shell
- **草稿自动保存** — 未发送的输入内容在页面刷新后自动恢复
- **外部链接** — Markdown 中的链接在系统浏览器中打开，而非 Electron 窗口内
- **完成提示音** — Claude 回复完成时播放提示音（可在设置中开关）
- **相对时间戳** — 消息显示"2分钟前"、"1小时前"等，实时更新，悬停显示绝对时间
- **GFM 任务列表** — `- [x]` 和 `- [ ]` 渲染为美观的复选框样式
- **消息书签** — 右键点击消息可添加书签，方便快速定位重要信息
- **字数/Token 估算** — 悬停助手消息可查看字数和大致 Token 数量
- **紧凑模式** — 减少间距以显示更多对话内容（设置 > 紧凑模式）
- **删除确认** — 删除会话需两次点击确认，防止误操作
- **滚动位置记忆** — 切换会话时自动保存和恢复滚动位置
- **复制为 Markdown** — 右键助手消息可复制原始 Markdown 源码
- **复制为富文本** — 右键助手消息可复制带格式的内容，粘贴到邮件和文档中保留样式
- **快捷键速查** — `Ctrl+/` 打开浮层显示所有可用快捷键
- **系统消息样式** — 错误和系统消息有独特的红色调外观
- **上下文输入指示器** — 显示 Claude 正在做什么："思考中..."、"执行命令..."、"读取文件..."等
- **会话数量徽章** — 侧边栏历史标签页显示会话总数
- **笔记数量徽章** — 侧边栏笔记标签页显示笔记总数
- **可折叠代码块** — 长代码块自动折叠，支持"展开/收起"切换
- **代码块行数** — 代码块标题栏显示行数
- **日期分隔线** — 消息按日期分组，显示"今天"、"昨天"或日期标题
- **专注模式** — `Ctrl+Shift+F` 隐藏侧边栏和终端，最大化聊天面板
- **书签面板** — 工具栏下拉列表显示所有书签消息，支持点击跳转
- **未读消息计数** — 向上滚动时，滚动到底部按钮显示新消息数量
- **双击复制** — 双击任何消息即可复制文本到剪贴板

- **全部折叠/展开** — 通过统计面板或 `Ctrl+Shift+C` 一键折叠或展开所有消息
- **原始 Markdown 切换** — 通过切换按钮查看助手消息的原始 Markdown 源码
- **工具栏消息计数** — 工具栏显示当前对话的总消息数量
- **滚动进度条** — 消息列表顶部显示细进度条，指示滚动位置
- **快速会话切换** — `Ctrl+[` / `Ctrl+]` 快速在会话间切换，无需使用侧边栏
- **图片灯箱预览** — 点击任何图片附件打开全屏预览，支持缩放、旋转和键盘控制
- **窗口标题通知** — 窗口标题显示会话名称，Claude 回复完成时闪烁 (*) 提示；窗口未聚焦时任务栏图标闪烁以吸引注意
- **原生桌面通知** -- 窗口未聚焦时 Claude 回复完成后弹出系统级通知；点击通知可聚焦窗口查看回复
- **Ctrl+K 清除** — 匹配终端习惯的清除对话快捷键
- **流式加载动画** — 流式响应期间工具栏显示旋转动画指示器
- **增强表格样式** — Markdown 表格支持圆角边框、强调色表头、悬停高亮
- **响应式侧边栏** — 窗口宽度低于 600px 时侧边栏自动折叠
- **持久化排序偏好** — 会话排序偏好（最新/最旧/字母序）跨重启保持

- **会话键盘导航** — 上/下方向键在侧边栏会话列表中导航
- **状态栏流式指示器** — 活跃响应期间显示脉冲绿点和"Streaming"标签
- **输入快捷操作** — 输入框上方的"Clear"、"@file"、"/cmd"快捷按钮
- **任务队列** — 预先排队多条指令，Claude 依次自动执行；支持随时暂停、恢复或清空队列

- **系统托盘** -- 最小化到系统托盘；全局热键 `Ctrl+Shift+Space` 随时召唤窗口；右键托盘可快速访问最近会话、切换主题、打开工作目录
- **剪贴板快捷操作** -- 在任何应用中按 `Ctrl+Shift+G` 可读取剪贴板内容并打开 AIPA 准备处理；非常适合快速翻译、总结、改写等场景
- **继续上次对话** -- 欢迎页面显示醒目的卡片，一键恢复最近的会话；显示会话标题，快速回到上次中断的工作
- **重新生成回复** — 一键重新生成 AI 回复，或使用 `Ctrl+Shift+R` 快捷键；移除上一次回复并重新发送提示
- **消息编辑** — 编辑任意已发送消息，对话将从该处截断并重新发送编辑后的内容以获取新回复

- **提示词模板** — 预配置的系统提示词角色（写作助手、研究分析师、导师、代码审查员、创意写作、效率教练），可在设置中下拉选择；支持创建自定义模板（最多 20 个），打造个人工作流库，如会议记录、邮件起草、周报模板等
- **复制对话** — 使用 `Ctrl+Shift+X` 或工具栏按钮将整个对话以 Markdown 格式复制到剪贴板；无需文件对话框，即时复制
- **ARIA 无障碍访问** — 地标角色（`application`、`banner`、`main`、`complementary`、`navigation`、`status`、`log`），`aria-live` 动态区域，所有交互元素均有 `aria-label`，支持屏幕阅读器
- **TypeScript 零错误** — `tsc --noEmit` 严格模式下零编译错误；包含 SpeechRecognition Web API 类型声明
- **会话标签** — 彩色标签分类系统（工作、个人、研究、调试、文档、归档），通过悬停菜单分配标签，侧边栏按标签过滤，标签名称可在设置中自定义
- **快捷笔记** — 侧边栏内置笔记本，可随时记录想法、待办或对话要点；自动保存、重启后保留、最多支持 100 条笔记
- **笔记与聊天互通** — 一键将笔记发送到聊天输入框（"发送到聊天"）；右键菜单将 AI 回复保存为笔记（"保存为笔记"）；无缝衔接笔记和 AI 对话

- **笔记标题自动生成** — 笔记内容首行为 Markdown 标题（`#`、`##`、`###`）时自动提取为笔记标题
- **笔记分类** — 支持创建最多 10 个彩色分类标签，按分类筛选笔记，在编辑器中为笔记指定分类，内联管理（创建、重命名、删除）分类
- **剪贴板快捷操作** — 聊天工具栏中的"粘贴并提问"按钮，一键读取剪贴板文字并提供快捷操作：摘要总结、翻译、改写润色、解释说明、语法检查；翻译功能自动识别目标语言
- **笔记导出** — 通过编辑器工具栏的下载按钮将单个笔记导出为 Markdown 文件（调用系统保存对话框）；通过笔记面板标题栏的"导出全部"按钮批量导出所有笔记（或按当前分类筛选）到指定文件夹
- **笔记导入** — 通过笔记面板标题栏的"导入"按钮导入 `.md` 和 `.txt` 文件为笔记；支持多文件选择，自动从 Markdown 标题或文件名提取笔记标题，遵守 100 条笔记上限

- **上下文窗口警告** — 上下文使用超过 80% 时显示可关闭的警告横幅，提供"新对话"快捷操作；超过 90% 时变为红色
- **输入 Token 估算** — 输入框显示估算 Token 数（约为字符数/4），与字数和字符数并排显示，帮助用户在发送前了解上下文用量
- **回复后自动聚焦输入框** — Claude 完成回复后输入框自动获得焦点，无需手动点击即可立即输入下一条消息
- **文本选择工具栏** — 选中消息中的任意文本后弹出浮动工具栏，支持复制、引用回复、保存到笔记三个快捷操作
- **Escape 停止生成** — 按 `Escape` 键立即停止 Claude 的回复生成；已收录在快捷键速查表中
- **时段问候语** — 欢迎页面根据当前时间显示"早上好/下午好/晚上好"，下方展示今日日期
- **内联标题重命名** — 点击聊天顶栏的会话标题即可内联重命名；按 Enter 保存，Escape 取消
- **持久化输入历史** — 已发送的消息通过 localStorage 跨页面刷新和应用重启保留（最多 50 条）
- **欢迎页最近提示** — 欢迎页面显示最近 3 条已发送的提示词，支持一键快速重用，可点击"清除"按钮重置历史
- **完整右键菜单** — 右键任意消息可执行全部操作：复制、引用回复、编辑（用户消息）、保存为笔记（AI 消息）、书签、折叠、评分、回退
- **引用预览横幅** — 引用消息时在输入框上方显示可视化预览，支持关闭按钮，发送时自动将引用文本以 Markdown 引用格式添加到消息前
- **会话消息数量徽章** — 会话列表行在时间戳旁显示消息数量，快速了解对话规模
- **欢迎页快捷键提示** — 欢迎页快捷操作按钮显示对应的键盘快捷键（如 `Ctrl+,`、`Ctrl+B`）
- **增强会话列表空状态** — 空会话列表显示友好的图标和提示文字，替代空白面板
- **笔记编辑器字数统计** — 笔记编辑器标题栏实时显示字数统计，与字符数并排显示
- **笔记阅读时间估算** — 笔记编辑器标题栏根据字数显示预计阅读时间（如 "约2分钟阅读"）
- **文件浏览器上级目录导航** — 文件浏览器标题栏的向上箭头按钮可直接导航到上级目录，无需使用文件夹选择对话框
- **文件类型图标** — 文件浏览器为 60+ 种文件类型显示彩色图标：代码文件（TypeScript、Python、Rust 等）、文档、图片、视频、音频、压缩包、数据库和配置文件

- **会话导出** — 通过侧边栏操作按钮中的下载图标，将任意会话导出为 Markdown 文件
- **多会话选择与批量删除** — 进入选择模式可勾选多个会话并一次性删除；支持全选/取消全选、底部浮动操作栏与确认提示，当前活动会话受保护不可删除

- **笔记 Markdown 工具栏** — 格式化工具栏提供加粗、斜体、标题、无序列表、有序列表、代码和链接按钮；可对选中文本添加 Markdown 语法或在光标处插入

## 安全

AIPA 遵循 Electron 安全最佳实践：

- **safeStorage 加密** — API 密钥通过操作系统密钥链加密存储（Windows 使用 DPAPI，macOS 使用 Keychain），无硬编码密钥
- **内容安全策略（CSP）** — 严格的 CSP 头部防止 XSS 和未授权外部资源加载
- **沙箱化渲染进程** — 启用 `sandbox: true`，渲染进程无法直接访问 Node.js
- **IPC 输入验证** — 所有文件系统 IPC 处理器对路径进行白名单验证
- **环境变量净化** — CLI 子进程仅接收明确白名单中的环境变量，防止密钥泄露

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/CipherSlinger/AIPA.git
cd AIPA/electron-ui

npm install

node_modules/.bin/electron dist/main/index.js

## 项目结构

```
AIPA/
├── package/          # 内置的 Claude Code CLI（只读，已打包）
└── electron-ui/      # Electron 应用
    ├── src/
    │   ├── main/     # 主进程（Node.js）：PTY、IPC、会话、配置
    │   ├── preload/  # 上下文桥接（将 electronAPI 暴露给渲染进程）
    │   └── renderer/ # React 界面（Vite + Tailwind）
    └── dist/         # 编译产物（构建生成，不提交到 git）

## 架构说明

应用通过两种模式与 Claude Code CLI 交互：

| 模式 | 用途 | 原理 |
|------|------|------|
| **PTY**（node-pty） | 终端面板 | 通过 ConPTY 启动 `node cli.js`，将原始终端 I/O 流传输给 xterm.js |
| **Stream-JSON** | 对话面板 | 启动 `node cli.js --input-format stream-json --output-format stream-json --print`，解析 NDJSON 事件流 |

渲染进程与主进程之间的所有通信通过 `window.electronAPI` 进行（定义于 `src/preload/index.ts`）。

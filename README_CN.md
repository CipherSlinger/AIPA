# AIPA — AI 个人助手

> 你的桌面智能伙伴。随问随答，随需随做——由 Claude 驱动。

AIPA 是一款基于 Electron + React 构建的桌面 AI 个人助手。它的目标不仅仅是一个聊天窗口，而是一个真正驻留在你桌面上的智能代理：它能理解你的文件与系统环境，编写并执行代码，浏览网页，从头到尾地完成现实世界中的任务。底层通过驱动 [Claude Code](https://claude.ai/code) CLI 作为执行引擎来实现这一切。

## 产品愿景

大多数 AI 工具是"回答机器"。AIPA 要做的是一个**行动者**。

- 你描述你要做什么，它负责想清楚怎么做。
- 它可以读取、创建、修改你本地的任何文件。
- 它可以运行 Shell 命令和脚本。
- 它可以访问网页、抓取信息。
- 它记住你的历史会话，随时继续上次的工作。
- 它将每一步操作透明呈现，你始终掌握最终决定权。

Claude Code CLI 是引擎，AIPA 是驾驶舱。

## 功能特性

- **对话面板** — 通过 stream-JSON 协议与 Claude 进行结构化对话，支持工具调用可视化
- **终端面板** — 基于 xterm.js 的完整 PTY 终端，直接运行 Claude Code
- **历史会话** — 浏览并恢复 `~/.claude/projects/` 中的历史对话，支持搜索高亮、骨架屏加载和当前会话高亮
- **文件浏览器** — 在侧边栏中浏览工作目录
- **设置面板** — 配置 API 密钥、模型、字体、工作目录、CLI 参数及 MCP 服务器
- **引导向导** — 首次启动时的图形化引导配置流程
- **消息评分** — 通过右键菜单对 AI 回复进行点赞 / 踩评价
- **消息右键菜单** — 右键点击任何消息可快速操作：复制、评分、回滚到检查点
- **思考块** — 可折叠的扩展思考过程展示
- **Token 用量与费用** — 状态栏实时显示上下文用量条、每轮 Token 计数和累计会话费用
- **权限提示卡片** — 以友好方式展示工具调用权限请求（Bash、写文件、编辑文件、网页访问等），支持一键允许 / 拒绝
- **Slash 命令弹窗** — 输入框中输入 `/` 弹出可用命令列表
- **@提及弹窗** — 通过 `@` 引用文件或上下文
- **计划卡片** — 在对话中以结构化卡片展示执行计划
- **图片粘贴** — 直接将图片粘贴至对话输入框
- **文件拖放** — 从系统文件管理器拖放文件到对话区域；图片自动附加，其他文件通过 `@路径` 引用
- **对话导出** — 通过工具栏按钮或 `Ctrl+Shift+E` 将对话导出为 Markdown 或 JSON
- **命令面板** — `Ctrl+Shift+P` 打开模糊搜索面板，快速访问所有操作和命令
- **键盘快捷键** — `Ctrl+N` 新建对话、`Ctrl+B` 切换侧边栏、`Ctrl+`` 切换终端、`Ctrl+L` 聚焦输入框、`Ctrl+,` 设置、`Ctrl+Shift+E` 导出、`Ctrl+F` 对话内搜索、`Ctrl+/` 快捷键速查
- **会话自动标题** — 首次对话后自动生成会话标题，侧边栏实时刷新
- **系统提示** — 为每次会话注入自定义系统提示词
- **智能自动滚动** — 自动滚动尊重用户意图；阅读较早消息时会显示"回到底部"按钮
- **生成计时器** — 实时显示 Claude 工作时长
- **设置关于页** — 版本信息、外部链接、快捷键参考、运行时信息、恢复默认设置
- **虚拟消息列表** — 使用 `@tanstack/react-virtual` 实现 100+ 消息时的流畅性能
- **RAF 节流流式传输** — 文本增量通过 `requestAnimationFrame` 批量处理，最小化 UI 开销
- **对话内搜索** — `Ctrl+F` 搜索当前对话，支持匹配高亮和前后导航
- **输入历史** — 按上/下方向键在历史发送记录中切换，类似终端 Shell
- **草稿自动保存** — 未发送的输入内容在页面刷新后自动恢复
- **外部链接** — Markdown 中的链接在系统浏览器中打开，而非 Electron 窗口内
- **完成提示音** — Claude 回复完成时播放提示音（可在设置中开关）
- **相对时间戳** — 消息显示"2分钟前"、"1小时前"等，实时更新，悬停显示绝对时间
- **GFM 任务列表** — `- [x]` 和 `- [ ]` 渲染为美观的复选框样式
- **消息书签** — 右键点击消息可添加书签，方便快速定位重要信息
- **字数/Token 估算** — 悬停助手消息可查看字数和大致 Token 数量
- **紧凑模式** — 减少间距以显示更多对话内容（设置 > 紧凑模式）
- **删除确认** — 删除会话需两次点击确认，防止误操作
- **滚动位置记忆** — 切换会话时自动保存和恢复滚动位置
- **复制为 Markdown** — 右键助手消息可复制原始 Markdown 源码
- **快捷键速查** — `Ctrl+/` 打开浮层显示所有可用快捷键
- **系统消息样式** — 错误和系统消息有独特的红色调外观
- **上下文输入指示器** — 显示 Claude 正在做什么："思考中..."、"执行命令..."、"读取文件..."等
- **会话数量徽章** — 侧边栏历史标签页显示会话总数
- **可折叠代码块** — 长代码块自动折叠，支持"展开/收起"切换
- **代码块行数** — 代码块标题栏显示行数
- **日期分隔线** — 消息按日期分组，显示"今天"、"昨天"或日期标题
- **专注模式** — `Ctrl+Shift+F` 隐藏侧边栏和终端，最大化聊天面板
- **书签面板** — 工具栏下拉列表显示所有书签消息，支持点击跳转
- **未读消息计数** — 向上滚动时，滚动到底部按钮显示新消息数量
- **双击复制** — 双击任何消息即可复制文本到剪贴板
- **会话置顶** — 在侧边栏中星标/置顶会话，置顶会话排在最前面
- **消息折叠** — 折叠单条消息以减少视觉干扰，折叠时显示内容预览
- **对话统计** — 工具栏弹出面板显示消息数、字数总计、工具使用次数和会话费用
- **全部折叠/展开** — 通过统计面板或 `Ctrl+Shift+C` 一键折叠或展开所有消息
- **原始 Markdown 切换** — 通过切换按钮查看助手消息的原始 Markdown 源码
- **工具栏消息计数** — 工具栏显示当前对话的总消息数量
- **滚动进度条** — 消息列表顶部显示细进度条，指示滚动位置
- **快速会话切换** — `Ctrl+[` / `Ctrl+]` 快速在会话间切换，无需使用侧边栏
- **图片灯箱预览** — 点击任何图片附件打开全屏预览，支持缩放、旋转和键盘控制
- **窗口标题通知** — 窗口标题显示会话名称，Claude 回复完成时闪烁 (*) 提示
- **Ctrl+K 清除** — 匹配终端习惯的清除对话快捷键

- **笔记标题自动生成** — 笔记内容首行为 Markdown 标题（`#`、`##`、`###`）时自动提取为笔记标题

## 安全

AIPA 遵循 Electron 安全最佳实践：

- **safeStorage 加密** — API 密钥通过操作系统密钥链加密存储（Windows 使用 DPAPI，macOS 使用 Keychain），无硬编码密钥
- **内容安全策略（CSP）** — 严格的 CSP 头部防止 XSS 和未授权外部资源加载
- **沙箱化渲染进程** — 启用 `sandbox: true`，渲染进程无法直接访问 Node.js
- **IPC 输入验证** — 所有文件系统 IPC 处理器对路径进行白名单验证
- **环境变量净化** — CLI 子进程仅接收明确白名单中的环境变量，防止密钥泄露

## 环境要求

- Windows 10/11 x64
- [Node.js](https://nodejs.org/) 18 或更高版本（需在 PATH 中）
- [Anthropic API 密钥](https://console.anthropic.com/)

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/CipherSlinger/AIPA.git
cd AIPA/electron-ui

# 2. 安装依赖
npm install

# 3. 构建
npm run build

# 4. 启动
node_modules/.bin/electron dist/main/index.js
```

首次启动后，点击侧边栏齿轮图标进入**设置**，填入你的 Anthropic API 密钥。

## 开发模式（热更新）

```bash
# 终端 1 — 构建主进程 & 启动 Vite 开发服务器
npm run build:main && npm run build:preload
npx vite

# 终端 2 — 以开发模式启动 Electron
NODE_ENV=development node_modules/.bin/electron dist/main/index.js
```

## 更新内置 CLI

`package/` 目录中包含内置的 Claude Code CLI。项目提供了两个辅助脚本，用于检测并更新它：

```bash
# Bash（Linux / macOS / Windows Git Bash）
bash update-cli.sh

# Windows 命令提示符
update-cli.bat
```

两个脚本均会将 `package/package.json` 中的当前版本与 npm 上的最新版本进行比对，并在下载替换 `package/` 前提示你确认。

## 项目结构

```
AIPA/
├── package/          # 内置的 Claude Code CLI（只读，已打包）
└── electron-ui/      # Electron 应用
    ├── src/
    │   ├── main/     # 主进程（Node.js）：PTY、IPC、会话、配置
    │   ├── preload/  # 上下文桥接（将 electronAPI 暴露给渲染进程）
    │   └── renderer/ # React 界面（Vite + Tailwind）
    └── dist/         # 编译产物（构建生成，不提交到 git）
```

## 架构说明

应用通过两种模式与 Claude Code CLI 交互：

| 模式 | 用途 | 原理 |
|------|------|------|
| **PTY**（node-pty） | 终端面板 | 通过 ConPTY 启动 `node cli.js`，将原始终端 I/O 流传输给 xterm.js |
| **Stream-JSON** | 对话面板 | 启动 `node cli.js --input-format stream-json --output-format stream-json --print`，解析 NDJSON 事件流 |

渲染进程与主进程之间的所有通信通过 `window.electronAPI` 进行（定义于 `src/preload/index.ts`）。

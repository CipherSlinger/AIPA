User explicitly stated (2026-03-27): "The iteration direction has leaned toward a programming AI agent, but what I want is a personal desktop AI assistant."

**Why:** The product has accumulated 100+ features heavily skewed toward developer workflows (code review, refactoring, command palette, terminal integration). The user wants AIPA to serve a broader audience with daily office tasks.

**How to apply:**
- When prioritizing features, favor general-purpose assistant capabilities (PPT generation, weekly reports, email drafting, schedule management) over developer-specific tools
- Reference Doubao (ByteDance's AI assistant) for skill/capability inspiration
- Reference OpenClaw and Claude Code official agent patterns for model capability design
- UI/UX decisions should optimize for non-technical users, not just developers
- Future PRDs should evaluate features through the lens of "personal assistant" not "coding tool"

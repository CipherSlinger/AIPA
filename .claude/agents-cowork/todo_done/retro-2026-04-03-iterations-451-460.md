## Action Items

### 1. Tester structural enforcement (P0 -- critical, 4th time flagged)
The "invoke tester every 5 iterations" rule has been in place since the 432-440 retro but only produced 1 checkpoint out of 6 expected (445 succeeded, 450/455/460 all missed). Voluntary reminder-based approach has failed.

**New approach**: Add a hard gate in the agent-leader workflow: before any commit at iteration N where N % 5 === 0, the leader MUST invoke aipa-tester. The commit cannot proceed until tester produces output. This is a blocking requirement, not an advisory one.

### 2. Prevent concurrent agent collision (P1)
Two agent instances working on the same PRDs caused iteration number collision and divergent implementations. Prevention: before starting an iteration, leader must check if background agents are running on the same codebase. If so, wait or coordinate.

### 3. Automate PRD archival (P1)
After 3 retros flagging manual PRD archival as a gap, implement a simple rule: the agent that commits an iteration that completes a PRD's last acceptance criterion MUST move the PRD to todo_done/ in the same commit.

### 4. useStreamJson.ts approaching threshold (P2)
At 590 lines, approaching the 600-line threshold. If further IPC event handling is added, consider extracting event handlers into a separate file (e.g., `useStreamJsonHandlers.ts` for the switch/case block).

### 5. ChatPanel.tsx growth monitoring (P2)
Grew from 492 to 542 lines due to branching features. If it continues growing, extract the branching-related state and handlers (handleFork, compareMode) into a `useBranching.ts` hook.

# PRD: 会话标签页 — 多会话并行视图

_版本：v1 | 状态：Draft | PM：aipa-pm | 日期：2026-04-06_

## 一句话定义

让用户同时打开多个会话标签页，像浏览器一样在对话之间快速切换，不再被迫丢失当前上下文。

## 背景与动机

AIPA 当前是单会话模式：用户在侧边栏点击另一个会话时，当前聊天被完全替换。这在以下场景造成严重摩擦：

1. **对比参考**：用户正在一个会话中编码，需要参考另一个会话中的解决方案 -- 切换过去就丢失了当前位置
2. **多任务并行**：同时进行代码审查、文档撰写、问题排查 -- 每次切换都是上下文中断
3. **长会话保护**：正在流式输出的会话，用户想查看历史会话 -- 不能在不中断当前流的情况下查看

所有主要竞品（Claude.ai、ChatGPT Desktop、Cursor、VS Code）均支持多标签页。AIPA 作为"桌面驾驶舱"，单会话限制是当前最大的结构性短板。

## 目标用户

**主要**：开发者 / 高频用户 -- 日常同时处理多个任务，需要快速在上下文间跳转
**次要**：知识工作者 -- 在不同话题的会话间做交叉引用

## 用户故事

作为一个使用 AIPA 进行多任务工作的用户，
我希望能够像浏览器标签页一样同时打开多个会话，
以便在不丢失任何会话上下文和滚动位置的前提下快速切换工作焦点。

## 功能范围

### In Scope（本版本包含）

1. **标签栏 UI**：在聊天区域顶部显示标签栏，每个打开的会话显示为一个标签
2. **标签管理**：从侧边栏双击或中键点击会话在新标签中打开；单击仍在当前标签中切换
3. **标签关闭**：每个标签显示关闭按钮（X），中键点击标签也可关闭
4. **标签切换快捷键**：Ctrl+Tab / Ctrl+Shift+Tab 切换标签；Ctrl+W 关闭当前标签

### Out of Scope（本版本不包含，说明原因）

- **拖拽排序**：标签拖拽重排序 -- 增加实现复杂度，可在 v2 迭代
- **分屏视图**：左右并排两个会话 -- 需要重构整个布局系统，属于独立 PRD
- **标签持久化**：重启后恢复上次打开的标签组 -- 需要修改 electron-store schema，可在 v2 迭代
- **超过 8 个标签的溢出处理**：初版限制最多 8 个标签，满时新开标签替换最旧的非活动标签

## 功能详述

### 功能 1：标签栏组件（TabBar）

**描述**：在 ChatPanel 顶部（ChatHeader 上方）渲染水平标签栏。每个标签显示会话标题（截断到 20 字符）和关闭按钮。活动标签有高亮底边框。

**交互逻辑**：
- 点击标签 → 切换到该会话（恢复消息列表和滚动位置）
- 点击标签关闭按钮 → 关闭该标签，切换到相邻标签
- 最后一个标签关闭时 → 显示 WelcomeScreen（新建会话状态）
- 标签栏为空时（初始状态）→ 不显示标签栏，保持当前单会话行为

**边界条件**：
- 会话正在流式输出时关闭标签 → 先 abort 流式传输，再关闭
- 同一会话不可重复打开 → 如果已在标签中，切换到已有标签
- 标签数达上限（8）→ 显示 toast 提示，不再打开新标签

**验收标准**：
- [ ] 标签栏在有 2+ 个标签时出现，仅有 1 个标签时隐藏（单会话体验不变）
- [ ] 每个标签显示会话标题（最多 20 字符 + 省略号）和关闭按钮
- [ ] 活动标签有视觉区分（高亮底边框 + 更深背景色）
- [ ] 标签栏支持横向滚动（当标签数较多时）

### 功能 2：会话状态隔离

**描述**：每个标签独立维护自己的消息列表快照和滚动位置。切换标签时恢复该会话的完整状态。

**交互逻辑**：
- 打开新标签 → 从 session:load ��载消息，存入 tab 的本地状态
- 切换离开标签 → 保存当前 scrollTop
- 切换回标签 → 恢复消息列表 + scrollTop
- 只有活动标签的 StreamBridge 接收流式事件

**边界条件**：
- 非活动标签的会话如果在 CLI 端有新消息（resume 场景）→ 下次切换到该标签时重新 load
- 内存保护：每个标签缓存的消息列表在内存中，8 个标签 × 大量消息可能占用较多内存 → 若单个标签消息超过 500 条，仅保留最新 200 条的快照，切回时重新 load

**验收标准**：
- [ ] 切换标签后，消息列表内容正确恢复
- [ ] 切换标签后，滚动位置恢复到之前的位置（误差 < 50px）
- [ ] 活动标签的流式消息正常接收和显示
- [ ] 非活动标签切换回时消息内容正确（含最新消息）

### 功能 3：标签操作快捷键

**描述**：提供键盘快捷键进行标签导航和管理。

**交互逻辑**：
- `Ctrl+Tab` → 切换到下一个标签（循环）
- `Ctrl+Shift+Tab` → 切换到上一个标签（循环）
- `Ctrl+W` → 关闭当前标签
- 侧边栏会话列表中键点击 → 在新标签中打开

**验收标准**：
- [ ] Ctrl+Tab / Ctrl+Shift+Tab 在标签间循环切换
- [ ] Ctrl+W 关闭当前活动标签
- [ ] 侧边栏中键点击会话在新标签中打开（不影响当前标签）

### 功能 4：标签与现有功能集成

**描述**：确保标签系统与现有功能（搜索、书签、导出等）正确配合。

**交互逻辑**：
- ChatHeader 的搜索、书签等功能始终作用于当前活动标签的会话
- 会话快速切换器（Ctrl+K）选择会话时，在当前标签中切换（不新开标签）
- 侧边栏单击会话行为不变（在当前标签中切换）

**验收标准**：
- [ ] 搜索（Ctrl+F）作用于当前活动标签
- [ ] 导出（Ctrl+Shift+E）导出当前活动标签的会话
- [ ] 书签面板显示当前活动标签的书签
- [ ] 新建会话（Ctrl+N）在当前标签中新建

## 非功能需求

- **性能**：标签切换应在 100ms 内完成（不含网络 load 时间）
- **内存**：8 个标签的内存增量不应超过 200MB（含消息缓存）
- **兼容性**：Windows 10/11

## 成功指标

- 用户平均同时打开标签数 > 2（表明功能被实际使用）
- 会话切换操作减少对侧边栏的依赖（通过标签切换替代）

## 优先级

- **P0**：标签栏 UI + 标签打开/关闭/切换 + 状态隔离
- **P1**：快捷键支持（Ctrl+Tab、Ctrl+W）
- **P1**：中键点击在新标签打开
- **P2**：标签溢出滚动

## 实现建议

核心思路：将 chatStore 中的消息/流状态从单实例提升为 `Map<tabId, TabState>`，活动 tabId 作为额外状态字段。TabBar 作为纯展示组件，从 store 读取 tab 列表和活动 tab。ChatPanel 根据活动 tabId 选择对应的 TabState 进行渲染。

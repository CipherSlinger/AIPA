# PRD：任务队列面板（Task Queue Panel）
_版本：v1 | 状态：Approved | PM：aipa-pm | 日期：2026-03-27_

## 一句话定义
让用户将多个任务预先排队，Claude 依次自动执行，无需用户手动触发每一条。

## 背景与动机
AIPA 定位是「doer not answer machine」。当前用户每次只能发送一条消息，等 Claude 回复后才能发下一条。对于批量任务（如「先分析这个文件，再写测试，再提交」），用户需要反复等待和手动操作。任务队列让 AIPA 真正像一个能持续工作的助手。

## 目标用户
开发者 Persona：需要让 Claude 连续执行多步任务，不想盯着屏幕等每一步完成。

## 用户故事
作为开发者，我希望预先输入多条任务指令并排队，以便 Claude 自动依次执行，我可以去做其他事情。

## 功能范围

### In Scope
- 在 Input 区域上方或侧边增加「队列」入口按钮
- 队列面板：显示待执行任务列表（可拖拽排序、删除单条）
- 用户可将输入框内容「加入队列」而非立即发送
- Claude 回复完成后自动发送队列中的下一条
- 队列状态：待执行 / 执行中 / 已完成 / 已跳过
- 支持「暂停队列」和「清空队列」
- 队列中可以有 1-20 条任务

### Out of Scope（v1）
- 队列持久化（刷新后队列清空）
- 条件分支（根据上一条结果决定下一条）
- 跨会话队列

## 功能详述

### 队列触发入口
**描述**：Input 区域增加「+ 加入队列」按钮（在发送按钮旁）
**交互逻辑**：点击后将当前输入内容加入队列，清空输入框，不发送
**验收标准**：
- [ ] 输入框有内容时「加入队列」按钮可用
- [ ] 点击后内容加入队列，输入框清空

### 队列面板
**描述**：可折叠的任务列表面板，显示在 Input 上方
**交互逻辑**：队列非空时自动显示；队列为空时收起
**验收标准**：
- [ ] 显示所有队列任务，每条有序号、内容预览、状态标签、删除按钮
- [ ] 可删除单条任务
- [ ] 显示「暂停」和「清空全部」按钮

### 自动执行逻辑
**描述**：Claude 完成当前回复后，自动发送队列头部的下一条任务
**交互逻辑**：监听 `cli:result` 事件，若队列非空且未暂停，自动发送下一条
**验收标准**：
- [ ] Claude 回复完成后自动发送下一条（无需用户操作）
- [ ] 已执行的任务状态变为「已完成」
- [ ] 队列全部完成后显示完成提示

### 队列控制
**验收标准**：
- [ ] 「暂停」按钮暂停自动执行（不影响当前进行的回复）
- [ ] 「恢复」按钮继续自动执行
- [ ] 「清空」按钮清空所有待执行任务（已完成的保留）

## 优先级
- P0：加入队列按钮、队列面板显示、自动执行
- P1：暂停/恢复/清空控制
- P2：拖拽排序

# PRD: Session Tags (会话标签)
_版本：v1 | 状态：Approved | PM：aipa-pm | 日期：2026-03-27_

## 一句话定义
为会话添加彩色标签分类系统，让重度用户可以按项目、主题或用途快速过滤和定位会话。

## 背景与动机
- AIPA 已有 108 次迭代，功能丰富（搜索、置顶、日期分组），但缺少「按用途分类」的能力
- 重度用户积累数十个会话后，仅靠搜索和置顶难以高效管理
- 竞品（ChatGPT Desktop、Cursor）已提供文件夹或项目分类概念
- 标签系统比文件夹更灵活（一个会话可以有多个标签），实现成本更低

## 目标用户
- 主要：每天使用 AIPA 进行多种任务（代码审查、写文档、调试）的开发者
- 次要：在多个项目间切换的自由职业者

## 用户故事
作为一个每天处理多个项目的开发者，
我希望能给会话打上项目标签（如 "AIPA"、"后端"、"文档"），
以便快速过滤只看某个项目的会话历史。

## 功能范围

### In Scope（本版本包含）
- 预设 6 个彩色标签（可自定义名称）
- 在会话列表右键菜单中为会话添加/移除标签
- 会话 item 上显示标签色点指示器
- 侧边栏标签过滤条（点击标签只显示对应会话）
- 标签持久化（electron-store）
- i18n 支持（en + zh-CN）

### Out of Scope（本版本不包含）
- 标签颜色自定义：使用预设 6 色即可，自定义颜色增加 UI 复杂度
- 标签嵌套/层级：过度设计，标签的核心价值在于扁平分类
- 会话拖拽排序：与现有排序系统冲突
- 标签在 NavRail 中独立入口：当前放在会话列表内足够

## 功能详述

### 标签数据模型
**描述**：6 个预设标签，每个有颜色和可编辑名称
**数据结构**：
```
tags: [
  { id: 'tag-1', name: 'Work', color: '#3b82f6' },     // blue
  { id: 'tag-2', name: 'Personal', color: '#22c55e' },  // green
  { id: 'tag-3', name: 'Research', color: '#f59e0b' },  // amber
  { id: 'tag-4', name: 'Debug', color: '#ef4444' },     // red
  { id: 'tag-5', name: 'Docs', color: '#8b5cf6' },      // purple
  { id: 'tag-6', name: 'Archive', color: '#6b7280' },   // gray
]
sessionTags: { [sessionId]: string[] }  // session -> tag IDs
```
**持久化**：通过 electron-store（prefsSet/prefsGet），与现有 prefs 系统一致
**验收标准**：
- [ ] 6 个预设标签有默认名称和颜色
- [ ] 标签名称可在 Settings 中编辑
- [ ] 标签分配关系持久化跨重启

### 会话标签分配
**描述**：通过右键菜单为会话添加/移除标签
**交互逻辑**：
- 右键会话 item → 弹出现有菜单 + 新增「Tags」子菜单
- Tags 子菜单显示 6 个标签选项，已分配的标签前有 checkmark
- 点击标签切换分配状态（toggle）
- 一个会话可以有多个标签
**验收标准**：
- [ ] 右键会话 item 显示 Tags 子菜单
- [ ] 可以为一个会话分配 1-6 个标签
- [ ] 再次点击已分配标签可以取消分配

### 标签视觉指示器
**描述**：会话列表 item 上显示已分配标签的色点
**交互逻辑**：
- 每个会话 item 的预览行右侧显示 1-3 个 8px 色点（超过 3 个显示 +N）
- 色点使用标签对应颜色
- Hover 色点显示标签名称 tooltip
**验收标准**：
- [ ] 有标签的会话显示对应颜色的色点
- [ ] 最多显示 3 个色点 + "+N" 溢出指示
- [ ] 色点 hover 显示标签名称

### 标签过滤条
**描述**：会话列表搜索框下方的标签过滤行
**交互逻辑**：
- 搜索框下方显示一行水平可滚动的标签 pill 按钮
- 每个 pill 显示标签颜色圆点 + 名称 + 该标签下会话数量
- 点击 pill 激活过滤，列表只显示包含该标签的会话
- 再次点击取消过滤
- 同时只能激活一个标签过滤
- 仅在有任何会话被标记时显示过滤行
**验收标准**：
- [ ] 标签过滤行显示在搜索框下方
- [ ] 各 pill 显示颜色点 + 名称 + 会话计数
- [ ] 点击 pill 过滤会话列表
- [ ] 再次点击取消过滤
- [ ] 水平可滚动
- [ ] 无已标记会话时不显示过滤行

### 标签管理（Settings）
**描述**：在 Settings 面板中管理标签名称
**交互逻辑**：
- Settings 面板新增「Tags」区域（在 Language 下方）
- 显示 6 个标签的颜色点 + 可编辑名称输入框
- 编辑名称后自动保存
**验收标准**：
- [ ] Settings 中显示 6 个标签的编辑行
- [ ] 每行有颜色指示和名称输入框
- [ ] 修改名称后自动持久化

## 非功能需求
- **性能**：标签过滤使用内存数据，无需额外 IPC 调用
- **可访问性**：标签 pill 按钮有 aria-label，色点有 aria-hidden
- **兼容性**：Windows 平台（现有唯一目标平台）

## 成功指标
- 用户可以在 3 秒内通过标签定位到特定项目的会话
- 标签系统不增加会话列表的渲染开销（数据在 store 内存中查找）

## 优先级
- **P0**：标签数据模型 + 持久化、右键分配标签、标签色点显示
- **P1**：标签过滤条、Settings 管理
- **P2**：i18n

## 依赖与风险
| 依赖项 | 负责方 | 风险等级 |
|--------|--------|----------|
| electron-store prefs 系统 | 现有基础设施 | 低 |
| SessionList 右键菜单 | 需要扩展现有菜单 | 低 |

## 后端需求
无。标签数据存储在 electron-store 的 prefs 中，与现有 prefsSet/prefsGet IPC 通道复用，不需要新的 IPC channel。

## 五、制作建议

### 录屏要点

| 要点 | 说明 |
|------|------|
| **分辨率** | 1920x1080 或 2560x1440，最终输出 1080p |
| **帧率** | 60fps 录制，最终输出 30fps（必要时可用慢动作） |
| **主题** | 全程使用深色主题，视觉效果更好，更符合技术产品调性 |
| **字体大小** | 编辑器/终端字体调到 16px 以上，确保视频压缩后可读 |
| **窗口尺寸** | AIPA 窗口占屏幕 80% 以上面积，避免周围桌面分散注意力 |
| **鼠标光标** | 使用高亮光标或点击水波纹效果，引导观众视线 |
| **打字速度** | 手动录入时控制在合理速度（约 3-4 字/秒），或使用自动打字工具 |
| **等待处理** | Claude 处理较慢的片段在后期加速（2-4x），保留关键输出 |

### 素材清单

| 类别 | 素材 | 来源 |
|------|------|------|
| **产品录屏** | AIPA 全部演示场景（约 6 段独立录屏） | 实际产品 |
| **Logo** | AIPA Logo（SVG/PNG，深色背景版本） | 项目资源 |
| **Motion Graphic** | 痛点对比图、安全信任图、功能图标动画 | 设计制作 |
| **GitHub 截图** | 仓库主页、Star 数、贡献者 | GitHub 实际截图 |
| **背景音乐** | 科技感、轻快节奏、无人声，适合 90 秒时长 | 免版权音乐库（如 Artlist、Epidemic Sound） |
| **音效** | 键盘敲击音、窗口弹出音、完成提示音 | 免版权音效库 |
| **字体** | 中文：思源黑体 / 苹方；英文：Inter / SF Pro | 开源字体 |

### 录屏场景准备

每个演示场景需要提前准备好数据和环境：

**场景 1（文件操作 + 代码执行）：**
- 准备一个包含多个 TODO 注释的示例项目
- 确保 Claude 能在 10 秒内完成文件扫描（文件不宜过多）
- 终端面板中的命令输出准备好测试用例

**场景 2（全局召唤）：**
- 提前在浏览器中打开一篇英文技术文章
- 确保系统托盘中 AIPA 已运行
- 测试 `Ctrl+Shift+G` 响应速度

**场景 3（AI 角色）：**
- 提前创建好 3 个不同的 Persona
- 确保切换过程流畅无卡顿

**场景 4（效率工具蒙太奇）：**
- 准备好几条示例笔记
- 确保技能市场有可安装的技能
- 命令面板中有足够的操作选项

### 后期制作建议

| 环节 | 建议 |
|------|------|
| **剪辑节奏** | 前 15 秒快节奏（每个镜头 2-4 秒），演示部分中等节奏（每段 12-16 秒），CTA 放慢 |
| **字幕** | 全���双语字幕（中文主字幕 + 英文副字幕），白底黑字或半透明背景 |
| **文字动画** | 核心功能名称用打字机效果浮现，品牌标语用渐入渐出 |
| **转场** | 场景间使用快速 wipe 或 zoom 转场，避免突兀的硬切 |
| **色彩校正** | 整体偏冷色调（蓝灰），符合科技产品气质，品牌蓝色 (#3b82f6) 用于强调元素 |
| **光标高亮** | 所有点击操作添加水波纹或放大动画，引导观众注意力 |
| **音乐混合** | 背景音乐在旁白时降低 30%，演示部分保持中等音量 |

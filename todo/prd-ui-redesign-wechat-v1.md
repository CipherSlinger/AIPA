# PRD: UI 重构 -- 类微信桌面端现代图形化风格
_版本：v1 | 状态：Draft | PM：aipa-pm | 日期：2026-03-26 | Iteration：53_

## 一句话定义

将 AIPA 的界面从当前的"工程师工具"风格重构为类微信桌面端的三栏布局 + 气泡对话风格，让 AI 对话体验从"操作终端"进化为"和智能助手聊天"，显著降低非技术用户的认知门槛。

## 背景与动机

### 为什么现在做

1. **产品成熟度拐点**：经过 52 次迭代，AIPA 的功能层已经相当完整（session 管理、tool use、file browser、export、search、bookmarks 等）。但视觉层仍停留在"开发者终端"的审美，与"AI Personal Assistant"的产品定位存在落差。

2. **用户扩展瓶颈**：当前 UI 对开发者友好，但对知识工作者、内容创作者等非技术用户形成使用壁垒。微信式布局是中国用户最熟悉的交互范式，采用这一范式可以把学习成本降到接近零。

3. **竞品压力**：ChatGPT Desktop 和 Claude.ai Web 都在朝更精致的对话式 UI 发展。AIPA 作为桌面原生应用，应在视觉品质上建立"本地应用比网页更好用"的认知。

4. **架构可行性**：现有 AppShell 已是 flexbox 二栏布局，Sidebar 已有 tab 切换机制。重构为三栏布局是对现有架构的自然扩展，不需要推翻重建。

### 现状分析

| 维度 | 当前状态 | 目标状态 |
|------|----------|----------|
| 布局 | 二栏（Sidebar + ChatPanel），Sidebar 内含 tab 切换 | 三栏（极窄导航 + 会话列表 + 主聊天区） |
| 消息展示 | 行式布局，头像在左，无气泡 | 气泡式，AI 左灰 / User 右色，双侧头像 |
| 导航 | Sidebar 顶部文字 tab | 独立图标导航栏 |
| 色调 | 深色为主，`--bg-primary` 单一背景 | 分层背景，导航深 / 列表中 / 聊天浅 |
| 会话列表 | 纯文本 item，无头像 | 圆形头像 + 标题 + 预览 + 时间 + 未读徽标 |

## 目标用户

- **主要 Persona**：知识工作者 -- 不写代码，但需要 AI 辅助写文档、分析数据、搜索信息。他们对微信的操作范式高度熟悉。
- **次要 Persona**：开发者 -- 已有用户，需确保重构不损害他们的效率（terminal panel、tool use 可视化等功能全部保留）。

## 用户故事

**US-1**：作为一个非技术用户，我希望打开 AIPA 后看到的界面和我熟悉的聊天软件类似（左边会话列表、右边对话），以便我能立即上手，不需要学习新的操作方式。

**US-2**：作为一个日常使用 AIPA 的用户，我希望 AI 的回复和我的消息在视觉上清晰区分（不同颜色的气泡、不同方向），以便我快速扫视对话历史时能一眼分辨出谁说了什么。

**US-3**：作为一个管理多个会话的用户，我希望在会话列表中看到头像、最后一条消息预览和时间，以便我无需点开会话就能判断要找的内容在哪。

## 功能详述

### 模块 1：三栏布局骨架（P0）

#### 描述

将现有 AppShell 的二栏布局（Sidebar + ChatPanel）重构为三栏布局：

```
+--------+---------------+----------------------------------+
| NavRail|  SessionPanel  |         ChatPanel                |
| 56px   |    240px       |         flex: 1                  |
| 固定    |   可拖拽调整    |                                  |
+--------+---------------+----------------------------------+
```

- **NavRail**（最左）：固定 56px 宽，深色背景（`#2e2e2e`），不可调整宽度
- **SessionPanel**（中间）：默认 240px，可拖拽调整（范围 180px - 400px），背景色 `#1e1e1e`（深色精致版）
- **ChatPanel**（最右）：弹性宽度 `flex: 1`，背景色 `#1a1a1a`（略深于 SessionPanel）
- **TerminalPanel**：保持现有行为，从右侧弹出，不影响三栏结构

#### 交互逻辑

- NavRail 始终可见，不参与 sidebar toggle 行为
- `Ctrl+B` toggle 的对象变为 SessionPanel（中间栏），NavRail 不受影响
- Focus mode (`Ctrl+Shift+F`) 隐藏 NavRail + SessionPanel，只留 ChatPanel
- 窗口宽度 < 600px 时自动收起 SessionPanel，NavRail 保留（可考虑 < 500px 也收起 NavRail）
- Title bar drag region 跨越三栏顶部，保持现有 32px 高度

#### 边界条件

- NavRail 在所有分辨率下保持 56px（不响应式收缩到更小值）
- SessionPanel 拖拽到最小值 180px 后继续拖拽不应导致布局错乱
- 三栏宽度分配持久化到 electron-store，重启后恢复

#### 验收标准

- [ ] 应用启动后可见三个明确的视觉分栏：极窄深色导航栏（约 56px）、中间会话列表栏（约 240px）、右侧聊天主区（剩余空间）
- [ ] 三栏之间有 1px 边框线分隔，视觉层次分明（导航最深、列表中、聊天最浅）
- [ ] 中间栏（SessionPanel）的拖拽调整宽度功能正常，范围 180px - 400px
- [ ] `Ctrl+B` 切换中间栏的显示/隐藏，NavRail 不受影响
- [ ] `Ctrl+Shift+F`（Focus mode）隐藏 NavRail 和 SessionPanel
- [ ] 窗口缩小到 600px 以下时，SessionPanel 自动收起
- [ ] Terminal panel 从右侧弹出时不破坏三栏布局
- [ ] 宽度设置持久化，重启应用后恢复上次的 SessionPanel 宽度

### 模块 2：左侧图标导航栏 -- NavRail（P0）

#### 描述

新增独立的极窄图标导航栏组件 `NavRail`，替代当前 Sidebar 顶部的文字 tab 栏。

#### 导航项定义

从上到下排列：

| 顺序 | 图标 | 标识 | 功能 | lucide-react 图标名 |
|------|------|------|------|---------------------|
| 1 | 对话气泡 | `chat` | 新建/当前对话（点击 = `Ctrl+N` 行为） | `MessageSquarePlus` |
| 2 | 时钟/历史 | `history` | 会话历史列表 | `History` |
| 3 | 文件夹 | `files` | 文件浏览器 | `FolderOpen` |
| 4 | 终端 | `terminal` | 切换 Terminal panel | `Terminal` |
| --- | 分隔 | --- | 弹性空白区域推到底部 | --- |
| 5 | 齿轮 | `settings` | 设置面板 | `Settings` |
| 6 | 头像 | `avatar` | 用户头像占位（圆形，后续可扩展为账户入口） | 纯色圆形 + `User` 图标 |

#### 选中状态

- 选中项：左侧出现 3px 宽的品牌色竖条（`var(--accent)` 或微信绿 `#07c160`），图标颜色变为白色/高亮色
- 未选中项：图标颜色为 `var(--text-muted)`（`#8b8b8b`），hover 时变为 `var(--text-primary)`
- 点击 `history` / `files` / `settings` 时，中间栏（SessionPanel）显示对应内容。如果中间栏已收起，点击导航项应同时展开中间栏
- 点击 `terminal` 不影响中间栏，而是 toggle 右侧 Terminal panel（与现有 `Ctrl+`` 行为一致）
- 点击 `chat` 触发新建对话流程（与现有 `Ctrl+N` 行为一致）

#### 底部头像

- 圆形，36px 直径，背景色 `var(--accent)` 或 `#5a3f8a`（品牌紫）
- 内部显示 `User` 图标（白色），后续迭代可替换为自定义头像
- 无点击行为（v1），后续可扩展为用户菜单

#### 边界条件

- History 导航项需显示 session count badge（继承现有行为）
- Tooltip 在 hover 时显示，延迟 500ms，内容为导航项名称
- 键盘导航：Tab 可聚焦到 NavRail，上下方向键在导航项之间移动

#### 验收标准

- [ ] NavRail 固定在窗口最左侧，宽度 56px，深色背景
- [ ] 包含 6 个图标项：Chat(新建)、History、Files、Terminal、Settings、Avatar（底部）
- [ ] 点击 History/Files/Settings 图标时，中间栏显示对应内容（与原 Sidebar tab 行为一致）
- [ ] 如果中间栏处于收起状态，点击 History/Files/Settings 会同时展开中间栏
- [ ] 选中的导航项左侧有品牌色竖条指示器，图标高亮
- [ ] 未选中项 hover 时图标变亮
- [ ] History 图标旁显示 session count badge
- [ ] 底部有圆形头像占位图标
- [ ] 点击 Chat 图标触发新建对话（与 `Ctrl+N` 行为一致）
- [ ] 点击 Terminal 图标 toggle Terminal panel（与 `Ctrl+`` 行为一致）

### 模块 3：消息气泡样式重构（P0）

#### 描述

将当前行式消息布局重构为对话气泡样式：
- **AI 消息**（assistant）：左对齐，浅灰气泡，左侧有头像
- **用户消息**（user）：右对齐，品牌色气泡，右侧有头像
- **系统消息**（system）：居中，无气泡，半透明文字

#### AI 消息（左侧）

```
[Avatar]  [     灰色气泡      ]
          [  消息内容...       ]
          [  时间戳  已读状态   ]
```

- 头像：40px 圆形，背景 `#5a3f8a`（品牌紫），内部 `Bot` 图标白色
- 气泡：
  - 背景色：`#2a2a2a`（深色版）— 比聊天区背景略浅，形成层次
  - 圆角：`border-radius: 2px 12px 12px 12px`（左上小圆角模拟微信尖角效果）
  - 最大宽度：`min(85%, 720px)`（防止全宽气泡，保持阅读舒适度）
  - 最小宽度：`60px`
  - 内边距：`12px 16px`
  - 文字颜色：`var(--text-primary)`（`#e0e0e0`）
- 气泡与头像间距：12px
- 气泡下方时间戳：`font-size: 11px`，颜色 `var(--text-muted)`，右对齐于气泡内

#### 用户消息（右侧）

```
          [   品牌色气泡     ]  [Avatar]
          [  消息内容...      ]
          [  时间戳           ]
```

- 头像：40px 圆形，背景 `var(--accent)`（品牌蓝 `#007acc`），内部 `User` 图标白色
- 气泡：
  - 背景色方案选择（见"配色方案"章节）：
    - **推荐方案 A（品牌蓝）**：`#264f78`（深蓝，与整体深色主题协调）
    - 备选方案 B（微信绿）：`#2d5a3a`（深绿色调暗版，适配深色主题）
  - 圆角：`border-radius: 12px 2px 12px 12px`（右上小圆角）
  - 最大宽度：`min(85%, 720px)`
  - 文字颜色：`#e8e8e8`（略亮于 AI 气泡文字，增强区分）

#### 系统消息

- 居中展示，无气泡，无头像
- 文字带半透明背景条：`background: rgba(244, 71, 71, 0.08)`（保留现有红色调）
- 圆角 4px，padding `4px 12px`，`font-size: 12px`

#### Tool Use 卡片（在气泡内）

- Tool use blocks 作为气泡内容的一部分，不单独成行
- 保留现有 ToolUseBlock 组件的功能，但视觉上简化为：
  - 内嵌在 AI 气泡中，用细线分隔
  - 背景比气泡稍深 `rgba(0,0,0,0.15)`
  - 工具名称和状态保留，但减少视觉噪音

#### Permission 卡片和 Plan 卡片

- 保持现有样式，不在本次重构范围
- 居中显示，不加气泡包裹（它们本身已有卡片样式）

#### 保留的功能

以下现有功能必须在新气泡布局中正常工作：
- 右键上下文菜单（MessageContextMenu）
- 双击复制
- hover 时的 Copy 和 Raw Markdown 按钮
- Thinking block 折叠/展开
- 图片附件展示和 lightbox
- 消息折叠（collapse）
- 消息搜索高亮（searchQuery）
- 消息入场动画
- 书签标记
- 消息评分

#### 边界条件

- 纯代码块消息：气泡宽度自动扩展到最大值（85%），代码块内部保持现有样式
- 超长单行文本：`word-break: break-word` 确保气泡不超出容器
- 图片消息：图片在气泡内展示，气泡宽度适配图片宽度（不超过最大值）
- 连续多条同角色消息：头像只在第一条显示，后续消息左侧留出头像宽度的空白对齐

#### 验收标准

- [ ] AI 消息显示为左侧灰色气泡，左侧有紫色圆形头像（Bot 图标）
- [ ] 用户消息显示为右侧品牌色气泡，右侧有蓝色圆形头像（User 图标）
- [ ] AI 气泡圆角左上为小圆角（尖角效果），用户气泡圆角右上为小圆角
- [ ] 气泡最大宽度不超过聊天区的 85% 且不超过 720px
- [ ] 系统消息居中显示，无气泡，带红色半透明背景
- [ ] 连续同角色消息，头像仅在首条显示，后续消息与首条气泡左（或右）对齐
- [ ] 消息时间戳显示在气泡内底部
- [ ] Tool use blocks 在 AI 气泡内部正常显示
- [ ] 右键菜单、双击复制、hover 按钮、thinking 折叠等现有功能正常工作
- [ ] 图片附件在气泡内展示，lightbox 点击正常
- [ ] 消息搜索高亮在气泡内正常工作
- [ ] 消息折叠（collapse）功能正常
- [ ] 消息入场动画正常
- [ ] Compact mode 下气泡间距适当缩小

### 模块 4：会话列表头像图标增强（P0）

#### 描述

为中间栏（SessionPanel）的会话列表 item 增加视觉辨识度，每个 item 添加圆角正方形头像图标。

#### Item 结构

```
+----+--------------------------------------------+
| 图 |  标题                         2m ago       |
| 标 |  最后一条消息预览文字...                     |
+----+--------------------------------------------+
```

- **头像图标**：36px 圆角正方形（`border-radius: 8px`），背景为基于会话 ID 或项目名生成的稳定色彩
  - 内部显示 `MessageSquare` 图标（白色）
  - Pinned 会话显示 `Star` 图标（替代原有纯 Star 小图标）
- **标题行**：会话标题（12px，`var(--text-primary)`） + 右侧时间戳（10px，`var(--text-muted)`）
- **预览行**：最后一条消息的前 50 字符预览（11px，`var(--text-muted)`），单行截断
- **选中态**：
  - 背景色：`rgba(7, 193, 96, 0.12)`（微信绿透明度）或 `rgba(0, 122, 204, 0.12)`（品牌蓝透明度）
  - 左侧边框：保持现有 3px accent 左边框
- **未读徽标**（预留）：头像图标右上角，红色圆点或数字，本次不实现后端逻辑，仅预留 UI 位置

#### 颜色生成算法

基于会话 ID（或项目名）的 hash 值，从预定义的 8 色调色板中选取：

```
#4a90d9  (蓝)
#50b86e  (绿)
#e67e22  (橙)
#9b59b6  (紫)
#e74c3c  (红)
#1abc9c  (青)
#f39c12  (黄)
#34495e  (灰蓝)
```

同一个 session 的头像颜色在所有展示中保持一致。

#### 边界条件

- 搜索高亮应在标题和预览文字上都生效
- 操作按钮（rename、fork、delete、pin）在 hover 时从右侧浮出，不覆盖头像
- Skeleton loader 需适配新的 item 结构（增加圆形头像 skeleton）

#### 验收标准

- [ ] 每个会话 item 左侧有 36px 圆角正方形头像图标
- [ ] 头像背景颜色基于会话 ID 稳定生成，同一会话颜色不变
- [ ] Pinned 会话的头像图标内显示 Star 图标
- [ ] 会话 item 包含标题行（标题 + 时间戳）和预览行（最后消息截断）
- [ ] 选中的会话 item 有品牌色背景高亮
- [ ] 搜索过滤时，标题和预览文字的匹配部分高亮
- [ ] hover 时操作按钮正常浮出，不遮挡头像
- [ ] Skeleton loader 适配新布局（包含头像占位）

## 配色方案选择

### 推荐：深色精致版

本次 Iteration 53 推荐采用**深色精致版**，原因：

1. **一致性**：现有 CSS 变量体系是深色主题，重构为深色精致版改动最小
2. **高级感**：深色分层背景（NavRail `#2e2e2e` > SessionPanel `#252525` > ChatPanel `#1a1a1a`）比纯黑更有视觉层次
3. **护眼**：桌面应用的主力使用场景是长时间工作，深色方案更适合
4. **品牌感**：品牌蓝 `#007acc` 在深色背景上更醒目

#### 色值定义（深色精致版）

| CSS 变量 | 色值 | 用途 |
|----------|------|------|
| `--bg-nav` | `#2e2e2e` | NavRail 背景 |
| `--bg-sessionpanel` | `#252525` | SessionPanel 背景 |
| `--bg-chat` | `#1a1a1a` | ChatPanel 背景 |
| `--bubble-ai` | `#2a2a2a` | AI 消息气泡 |
| `--bubble-user` | `#264f78` | 用户消息气泡（品牌蓝） |
| `--bubble-ai-text` | `#e0e0e0` | AI 气泡文字 |
| `--bubble-user-text` | `#e8e8e8` | 用户气泡文字 |
| `--nav-active-indicator` | `var(--accent)` | NavRail 选中指示条 |
| `--session-active-bg` | `rgba(0, 122, 204, 0.12)` | 选中会话背景 |

浅色主题方案（`--bg-chat: #ededed` 等）留到 P2 阶段，作为主题切换功能的一部分实现。

## 非功能需求

### 性能

- 气泡布局重构不应导致 MessageList 的虚拟化滚动性能下降（现有 `@tanstack/react-virtual` 集成必须保持正常）
- NavRail 作为静态组件，不应在消息流更新时触发重渲染
- 连续同角色消息的头像合并逻辑需在 render 层面高效处理，避免 O(n^2) 的上下文扫描

### 安全

- 无新的安全需求。本次重构纯视觉层面，不涉及 IPC、文件系统或网络请求。

### 可访问性
